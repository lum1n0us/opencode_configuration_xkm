# Static Analyze WAMR

## Overview

This skill runs clang-tidy static analysis on WebAssembly Micro Runtime (WAMR) projects. It intelligently performs either focused analysis on modified files (when changes.diff exists) or global scans of all source code, using compile_commands.json files generated by the build-wamr skill.

**Recommended Usage**: Better use it with build-wamr together for optimal workflow integration and consistent build artifacts.

## Prerequisites

### 1. Build Artifacts Required
Must have build outputs from build-wamr skill in inter_dir:
- `build/iwasm/compile_commands.json` - For iwasm component analysis
- `build/wamrc/compile_commands.json` - For wamrc component analysis

### 2. clang-tidy and clang-tidy-diff Required
Both clang-tidy and clang-tidy-diff must be installed and available in PATH. The skill will check for availability and stop if either is not found.

### 3. Configuration File Required
Must have `.clang-tidy` configuration file in:
- Repository root, OR
- Current working directory

## Quick Start

Run static analysis on built WAMR components:

```bash
python scripts/static_analyze.py <inter_dir> [--repo_path <path>]
```

**Parameters:**
- `inter_dir`: Directory containing build outputs and analysis files
- `--repo_path`: Path to WAMR repository (defaults to current directory)

## Analysis Modes

### 1. Focused Analysis (Recommended)
When `changes.diff` exists in inter_dir:
- **Method**: Uses `clang-tidy-diff -p 1 --path <build_dir> < changes.diff`
- **Scope**: Analyzes only modified lines from the diff file
- **Per-Component**: Runs separately for iwasm and wamrc build directories
- **Benefits**: Fastest execution, focuses on new potential issues in changed code
- **System Headers**: Automatically filtered out

### 2. Global Analysis  
When no `changes.diff` found:
- **Method**: Uses regular `clang-tidy` to scan all source files
- **Benefits**: Comprehensive code quality assessment of entire codebase
- **System Headers**: Automatically filtered using header-filter regex
- **Usage**: Suitable for full codebase audits and baseline quality checks

## Workflow Integration

**⚠️ Important**: Better use it with build-wamr together for optimal workflow integration and consistent build artifacts.

### Complete PR Analysis Pipeline (Recommended)

```bash

python scripts/static_analyze.py ./build_output --repo_path /path/to/wamr
```

## Configuration

### .clang-tidy Example
Create `.clang-tidy` in repository root:

```yaml
Checks: >
  bugprone-*,
  clang-analyzer-*,
  performance-*,
  readability-*,
  -readability-magic-numbers,
  -bugprone-easily-swappable-parameters

WarningsAsErrors: ''

HeaderFilterRegex: '(core/iwasm|wamr-compiler)/.*'

CheckOptions:
  - key: readability-identifier-naming.VariableCase
    value: lower_case
  - key: readability-identifier-naming.FunctionCase  
    value: lower_case
```

## Commands Used

The skill executes different commands based on analysis mode:

### Focused Analysis Commands
For each component (iwasm/wamrc):
```bash
clang-tidy-diff -p 1 -path ${inter_dir}/build/iwasm < ${inter_dir}/changes.diff

clang-tidy-diff -p 1 -path ${inter_dir}/build/wamrc < ${inter_dir}/changes.diff
```

**Note**: clang-tidy-diff uses `.clang-tidy` config file from repository root automatically.

### Global Analysis Commands
```bash
clang-tidy source_files... -p ${inter_dir}/build/iwasm \
  --config-file=.clang-tidy \
  --header-filter='^(?!.*(usr/include|/usr/local|/opt|system)).*'
```

## System Header Filtering

The skill automatically filters out issues reported in system headers to focus on project-specific code:

**Filtered Paths:**
- `/usr/include/` - System headers (Linux)
- `/usr/local/include/` - Local system headers
- `/opt/` - Optional software installations
- `/Applications/Xcode.app/` - macOS system headers
- `C:\\Program Files` - Windows system headers
- `/Library/Developer/` - macOS SDK headers

**Implementation:**
- **Focused Analysis**: clang-tidy-diff inherently focuses on changed lines
- **Global Analysis**: Uses `--header-filter` regex to exclude system paths
- **Output Parsing**: Additional filtering during issue extraction

## Error Handling

### Prerequisite Validation
- **Missing build directory**: Asks to run build-wamr first
- **No compile_commands.json**: Specific paths expected
- **Missing .clang-tidy**: Shows expected locations
- **clang-tidy not installed**: Stops execution, asks for installation
- **clang-tidy-diff not installed**: Stops execution, asks for installation

### Analysis Errors
- **File parsing failures**: Warnings with continued execution
- **clang-tidy errors**: Captured in output with warnings
- **Empty analysis**: Graceful handling with appropriate messaging

### Common Error Messages

**Missing Prerequisites:**
```
FAIL
Build directory not found: /path/to/inter_dir/build. 
Please run the build-wamr skill first to generate build artifacts.
```

**Missing Configuration:**
```
FAIL
.clang-tidy configuration file not found. Expected locations:
  - /repo/root/.clang-tidy
  - /current/dir/.clang-tidy
Please create a .clang-tidy configuration file with your desired rules.
```

**Missing Tools:**
```
FAIL
clang-tidy-diff is not installed or not available in PATH. 
Please ensure both clang-tidy and clang-tidy-diff are installed and available.
```
