# Analyze PR

## Overview

This skill analyzes GitHub Pull Request changes by classifying modified files into component categories based on the repository's existing `AGENTS.md` configuration file and generates comprehensive analysis reports with risk assessment and recommendations.

## Main Functionality

1. **Change Type Classification**: Determines if PR changes core components code - marks as "CODE CHANGE" or "NON-CODE CHANGE" in analysis.md
2. **Risk Assessment**: Calculates risk factor based on modified files and writes score to analysis.md
3. **Status Output**: Outputs analyze_pr_status.md indicating SUCCESS or FAIL
4. **Analysis Report**: Outputs analysis.md with all results from above steps

## Prerequisites

### 1. Fetch-PR Output Required
This skill works on data generated by the `fetch-pr` skill and expects these files in the inter_dir:
- `report.md` - PR metadata and comments
- `changes.diff` - Complete diff of all changes

### 2. AGENTS.md Configuration Required
The repository **must** contain an existing `AGENTS.md` file in the root directory that defines directory classifications. Every repository should have one. The skill will not create this file - it must already exist with proper project structure definitions.

## Risk Assessment Rules

- **Document modifications only** → low risk
- **Core components code (including headers) modified** → high risk
- **Test cases, samples, utilities only** → low risk
- **Large amount of code modified** → high risk
- **Build system, CI only** → medium risk

## Critical Limitations

- **ALWAYS** reads project structure classification from AGENTS.md - never assumes
- AGENTS.md defines which directories are core components
- Only *.c, *.h, *.cc files are considered "code" (not *.py, *.bash, etc.)
- If a directory can't be classified using AGENTS.md info → STOP and ask user for clarification

## Required Outputs

- **analysis.md**: Contains CODE CHANGE/NON-CODE CHANGE classification, risk score, and detailed analysis
- **analyze_pr_status.md**: Contains SUCCESS or FAIL status

## Quick Start

Run after using the fetch-pr skill:

```bash
python scripts/analyze_pr.py <pr_number> <inter_dir> [--repo_path <path>]
```

**Parameters:**
- `pr_number`: GitHub PR number (must match fetch-pr data)
- `inter_dir`: Directory containing fetch-pr output files
- `--repo_path`: Path to git repository (defaults to current directory)

## AGENTS.md Requirements

The skill reads directory classifications from the repository's existing `AGENTS.md` file. The file should contain sections defining project structure:

### Expected Format

```markdown
# Project Structure Classification

## Core Components
- src/
- lib/
- core/

## APIs
- api/
- include/
- public/

## Tests
- test/
- tests/
- spec/

## Documentation
- docs/
- doc/

## Build System
- CMakeLists.txt
- Makefile
- package.json

## Utilities
- utils/
- tools/
- scripts/
```

### Supported Section Names
The skill recognizes these section headers (case-insensitive, with # or ## markers):
- **Core Components** / **Core**
- **APIs** / **API**
- **Internal Headers** / **Internal**
- **Tests** / **Test**
- **Samples** / **Sample**
- **Documentation** / **Docs**
- **CI/CD** / **CI**
- **Build System** / **Build**
- **Utilities** / **Utils**

**Note**: The skill does not assume any default classifications. All directory patterns must be explicitly defined in the repository's `AGENTS.md` file.

## Output Files

### 1. Status Report (`analyze_pr_status.md`)
**Success format:**
```
SUCCESS
```

**Failure format:**
```
FAIL
AGENTS.md not found in repository root (/path/to/repo)
```

### 2. Analysis Report (`analysis.md`)
Comprehensive analysis including:
- **PR Summary**: Basic statistics and metadata
- **Change Type Classification**: CODE CHANGE or NON-CODE CHANGE based on *.c, *.h, *.cc files
- **Change Classification**: Files categorized by component type from AGENTS.md
- **Risk Assessment**: LOW/MEDIUM/HIGH risk rating with numerical score and rationale
- **File Type Analysis**: Breakdown by file extensions
- **Recommendations**: Actionable review guidance

## Classification Logic

The skill classifies each changed file by:
1. Reading directory patterns from `AGENTS.md`
2. Matching file paths against defined patterns
3. Supporting multiple classifications per file
4. **Validation**: Stopping and asking for clarification if any files cannot be classified

### Code Change Classification
- **CODE CHANGE**: Any *.c, *.h, *.cc files are modified
- **NON-CODE CHANGE**: Only non-code files (*.py, *.md, *.bash, etc.) are modified

### Risk Assessment Factors
Based on the following rules:
- Document modifications only: LOW risk (score 1)
- Core components code modified: HIGH risk (score 6+)
- Test/samples/utilities only: LOW risk (score 1)
- Build system/CI only: MEDIUM risk (score 3)
- Large code changes (>500 lines): HIGH risk (score 6+)
- Medium code changes (>200 lines): MEDIUM-HIGH risk (score 4+)

Additional factors:
- Core component changes: +2 risk
- API changes: +1 risk
- Build system changes: +1 risk

**Risk Levels:**
- HIGH: Score ≥ 5
- MEDIUM: Score 3-4
- LOW: Score < 3

## Template Customization

### Analysis Report Template
Customize the report format by editing `assets/analysis_template.md`. Supported placeholders:

**Basic Information:**
- `{pr_number}`, `{analysis_date}`, `{total_files}`
- `{lines_added}`, `{lines_deleted}`
- `{change_type}` - CODE CHANGE or NON-CODE CHANGE

**Analysis Sections:**
- `{classification_summary}` - High-level category counts
- `{detailed_classification}` - File-by-file breakdown
- `{risk_level}`, `{risk_score}`, `{risk_assessment}` - Risk analysis with numerical score
- `{file_types}`, `{change_patterns}` - Code quality metrics
- `{recommendations}` - Generated recommendations

## Usage Examples

**Basic analysis:**
```bash
# After running fetch-pr
python scripts/fetch_pr.py 123 ./pr_data
python scripts/analyze_pr.py 123 ./pr_data
```

**Custom repository path:**
```bash
python scripts/analyze_pr.py 456 ./analysis --repo_path /path/to/repo
```

**Batch analysis workflow:**
```bash
for pr in 100 101 102; do
  mkdir -p ./analysis/pr_$pr
  python scripts/fetch_pr.py $pr ./analysis/pr_$pr
  python scripts/analyze_pr.py $pr ./analysis/pr_$pr
done
```

## Error Handling

Common failure scenarios:
- **Missing AGENTS.md**: Error message asks to ensure the file exists in repository root
- **Empty AGENTS.md**: Error if no directory classifications found
- **Unclassifiable files**: Stops and asks user for clarification on how to classify files not covered by AGENTS.md
- **Missing fetch-pr files**: Validates required input files exist
- **Invalid diff format**: Handles malformed diff files gracefully
- **Repository access issues**: Clear error messages for path problems

**Important**: The skill will **not** create or provide templates for AGENTS.md. Every repository is expected to already have this configuration file with project-specific directory definitions.

All errors are captured with descriptive messages in the status report.
